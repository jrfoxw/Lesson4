Client/Servers

A Client communicates with a server via sending a get or post command which is then used by the server to respond with the contents of what was asked for.
Example:

  1. Client communicates with server via a protocol like HTTP.
  2. The Client uses a get request for whatever object it is requesting.
  3. Server takes the request and returns the object that the client has requested.
  4. GET shows up in urls, as name=value
  5. POST does not show up in url, and is used for sending to server secure information
  6. RESPONSES 200, 302, 404, 500 indicate the status of the request that was sent to the server
    1. 200 OK
    2. 302 FOUND
    3. 404 NOT FOUND
    4. 500 SERVER ERROR

Validation

Validation is extremely important in this day and age, especially on the web. Without Validation there would be no security on websites. Validating a users input is important for several reasons.

  1. Users can enter malicious code which can change the way a website works
  2. Import information stored on a server (credit card info, SSN, ect) could easily be compromised
  3. Without proper authentication and validation the web would not be a safe place.
  4. The user receives a better experience (UX) knowing that their information is safe and well guarded on the site.

HTML Templates

I've used python/Django several times in the past, and I can say, without a doubt that templating is probably the most powerful way to save a ton of time, or rather keep yourself DRY. HTML templates can make what would normally be a very difficult job much easier to complete.

  1. You create one base template and then extend on that
  2. It limits the amount of html/code that you have type
  3. To change headers or footers all you do is change the base template.
